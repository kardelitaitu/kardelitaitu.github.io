System Architecture

Explore how the modular core integrates with the white-label custom script engine.

High-Performance Rust Core

At the heart of oz-pos is a compiled Rust engine. Rust guarantees memory safety, zero-cost abstractions, and sub-millisecond execution speeds for critical checkout routines. The core manages inventory indexing, sqlite-backed transactions, local cashier security, and database write queues.

Lua Customization Layer

Rather than recompiling the system for every white-label custom client, oz-pos embeds a sandboxed Lua interpreter. This interpreter loads local config.lua scripts to modify receipt headers, custom branding colors, and dynamically override billing or pricing logic (like promotions and tax rates) instantly at runtime.

Offline-First Cloud Sync

Our cloud synchronization engine works background threads. If network connectivity is lost, sales queues are stored securely in local SQLite databases. Once internet connection returns, the system automatically uses a lightweight delta sync protocol to upload logs and update stock levels without cashier interruptions.
